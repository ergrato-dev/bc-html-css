# Proyecto Semana 02 — Mi CV en HTML

## 🎯 Objetivo

Crear un curriculum vitae personal en HTML semántico puro, sin CSS frameworks, aplicando todos los conceptos de la semana: estructura HTML5, etiquetas semánticas, headings, listas, enlaces e imágenes.

## 📋 Descripción

Construirás una página HTML de tu CV personal con las siguientes secciones:

1. **Encabezado** — nombre, título profesional y foto
2. **Sobre mí** — párrafo de presentación
3. **Experiencia laboral** — lista de posiciones con fechas
4. **Educación** — lista de estudios
5. **Habilidades** — lista de competencias técnicas y blandas
6. **Contacto** — enlaces de correo, teléfono y redes profesionales

## 📁 Estructura de archivos

```
3-proyecto/
├── starter/
│   └── index.html        ← Tu punto de partida (con TODOs)
└── solution/             ← Solo para instructores (oculto)
```

## 🚀 Instrucciones

1. Abre `starter/index.html` en VS Code
2. Lee cada `TODO` y completa la sección correspondiente
3. Verifica el resultado en el navegador con Live Server
4. Valida tu HTML en [https://validator.w3.org/](https://validator.w3.org/)

## 📌 Criterios de evaluación

| Criterio | Puntos |
|----------|--------|
| Estructura HTML5 correcta (DOCTYPE, head, body) | 10 pts |
| Uso correcto de etiquetas semánticas (header, main, footer, section) | 20 pts |
| Jerarquía de headings lógica y sin saltos | 15 pts |
| Listas apropiadas (ul/ol/dl) según el contenido | 15 pts |
| Todos los enlaces con atributos de seguridad y accesibilidad | 20 pts |
| Imagen con alt descriptivo y figure/figcaption | 10 pts |
| Validación W3C sin errores | 10 pts |
| **Total** | **100 pts** |

## ✅ Checklist de entrega

- [ ] El archivo se abre correctamente en el navegador
- [ ] No hay errores en la consola del navegador
- [ ] La página pasa el validador W3C sin errores
- [ ] Todas las imágenes tienen `alt`
- [ ] Los enlaces externos tienen `rel="noopener noreferrer"`
- [ ] Un solo `<h1>` por página

## 💡 Tips

- Usa `lang="es"` en la etiqueta `<html>` si el contenido es en español
- Guarda las imágenes en una carpeta `assets/images/` dentro del proyecto
- Puedes usar imágenes de [UI Avatars](https://ui-avatars.com/) como placeholder de foto
